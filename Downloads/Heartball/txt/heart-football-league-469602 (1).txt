* Heart Football League Project Architecture:
The Heart Football League (HfL) project is a comprehensive sports simulation built with a modern, modular architecture. It is designed to be a cross-platform web application that can run on both desktop and mobile devices, with a focus on a mobile-first user experience.Core ComponentsThe project is structured into three main components that work together to create the full application experience.Python Backend (Core Simulation):Purpose: This is the brain of the HfL universe. It contains all the core game logic, player definitions (player.py), game simulations (game.py), and systems for managing data like player contracts, injuries, and league dynamics.Technology: Python. The entire simulation engine is written in Python, allowing for complex and detailed game mechanics.Deployment: The Python backend is designed to be run in a variety of environments, including a web server (such as FastAPI) for a full-scale application or locally for testing. A key design principle is to make it runnable in a browser environment using a technology like Pyodide, enabling an offline-first, portable experience.Web Frontend (User Interface):Purpose: The user-facing part of the application. It provides dashboards, user interfaces for managing teams, viewing stats, and interacting with the game world. This includes the dashboard/index.html and its associated static assets.Technology: React. The main application (src/App.js and src/components/) is a Single Page Application (SPA) built with React for a dynamic and responsive experience.Deployment: The frontend is served as a static site, intended for deployment on a service like Firebase Hosting or Vercel.Firebase (Data Persistence & Cloud Services):Purpose: A real-time, scalable backend-as-a-service (BaaS) that handles data storage, user authentication, and real-time updates. It acts as the bridge between the Python simulation and the web frontend.Services Used:Firestore: Used for storing persistent game data (player rosters, league structure, user profiles) in a NoSQL document database. This allows for real-time synchronization between users and across devices.Firebase Authentication: Handles user sign-in and management.Firebase Hosting: Serves the static web application files.Firebase Functions: Can be used in the future to run server-side Python code that interacts with the simulation engine.Data Flow and InteractionThe system is designed with a clear separation of concerns:The Web Frontend interacts directly with Firebase Firestore. It reads data to display dashboards and writes data based on user actions (e.g., updating a team roster).The Python Backend is the source of truth for all game simulation logic. When the user initiates a game event (e.g., "Simulate Week 1"), a function or service on the frontend would trigger the Python engine.The Python Engine reads the necessary data from Firestore, runs the simulation, and then writes the updated game state back to Firestore. This ensures that all changes are persistent and immediately reflected across all user devices.This architecture ensures a performant, scalable, and highly available application, with the core logic separated from the presentation layer. It also leverages modern cloud services to handle the complexities of data management and deployment.




Heart Football League (HfL)Welcome to the Heart Football League project! This is a modern, web-based football simulation game designed with a mobile-first approach. It combines a powerful Python-based simulation engine with a dynamic React-based frontend and a Firebase backend for real-time data persistence.Project StructureThe project is organized into the following key directories:ARCHITECTURE.md: A high-level overview of the project's design and technology stack.README.md: This file! Contains project information and setup instructions.config/: Configuration files for the project.dashboard/: Static HTML, CSS, and JS files for a lightweight, mobile-friendly dashboard.data/: JSON and Python files containing core game data (e.g., player templates, league structure).public/: Public-facing static assets for the React app.roles/: Python scripts defining various "roles" or personas for the AI, media, and other systems.src/: The main source code for the React frontend, including components, services, and a python directory for the simulation engine.systems/: Modular Python systems for specific gameplay features, like Ultimate Team and Daily Fantasy.tests/: Unit and integration tests to ensure the game works as expected.Getting StartedFollow these steps to set up and run the project locally.PrerequisitesPython 3.x: Installed on your system.Node.js & npm: Installed on your system.Google Cloud SDK: Authenticated and configured for your project.gcloud auth logingcloud config set project [YOUR_PROJECT_ID]1. Project SetupFirst, ensure your project directory is correctly set up. You can use the create_hfl_project.py script provided earlier.2. Configure FirebaseNavigate to your Firebase console and create a new project. You'll need the following information to update your configuration files:Project IDAPI KeyAuth DomainStorage Bucket3. Install DependenciesFrom the project root directory, run the following commands to install the necessary packages for both the Python and Node.js parts of the project:# Install Node.js dependencies for the frontend
npm install


# Install Python dependencies for the simulation
pip install -r requirements.txt # (assuming you have a requirements.txt file, or install manually)
4. Run the ProjectYou can run the frontend and backend components separately.To run the React frontend:npm run dev
This will start a local development server, typically at http://localhost:3000.To run the Python simulation:python main.py
This will execute the main game loop or other core logic defined in main.py.5. DeploymentDeployment is handled using Firebase. The deploy-web.sh script is designed to build and deploy the application to Firebase Hosting../deploy-web.sh
This will build your React application and upload the static files to your Firebase project.Have fun building the Heart Football League!
